Installation

CuRobo is a library built in python with core compute components implemented as CUDA kernels and wrapped within python through pyTorch. We can install CuRobo for use directly through python following instructions for library installation. If you want to use CuRobo in Isaac sim, skip library instructions and follow Isaac sim instructions.

  • Library Installation

  • Install for use in Isaac Sim

  • Install nvblox (optional)

  • Generate Documentation

Library Installation

  1. Start with a pytorch docker or a python environment with pytorch >= 1.10.

  2. Clone CuRobo repository git clone https://github.com/NVlabs/curobo.git and run pip install -e . --no-build-isolation. This will take 20 minutes to install.

  3. Run python3 -m pytest . to verify that all unit tests pass.

  4. Look at python scripts in examples/ path for examples.

Note

NVIDIA Isaac Sim is not a required dependency of CuRobo. All functions of CuRobo can be accessed through python

Install for use in Isaac Sim

  1. Install NVIDIA Isaac Sim 2022.2.1 following isaac_sim_workstation_install

  2. Install CUDA 11.7 following cuda_instructions and add to your path with: export PATH=/usr/local/cuda-11.7/bin${PATH:+:${PATH}}

  3. Add an alias to Isaac Sim’s python in your bashrc file: echo "alias omni_python='~/.local/share/ov/pkg/isaac_sim-2022.2.1/python.sh'" >> ~/.bashrc `` and source it with ``source ~/.bashrc. If your isaac sim path is different, change the path accordingly.

  4. Clone this repository and run omni_python -m pip install -e . --no-build-isolation, where omni_python refers to ./python.sh from isaac sim installation (info).

  5. Run omni_python examples/isaac_sim/motion_gen_reacher.py to run a motion generation example. Once Isaac Sim loads, click play and move the target cube to start generating motions.

Note

Isaac Sim 2023.1: Isaac Sim examples have not been tested on 2023.1 and may not work. However, python functions will work in Isaac Sim 2023.1. We found these additional packages to be missing in 2023.1, install them with omni_python -m pip install tomli wheel before installing curobo.

Install nvblox (optional)

Installing as a Library

  1. Install nvblox with the ABI-compatibility mode

git clone https://github.com/valtsblukis/nvblox.git
sudo apt-get install -y libgoogle-glog-dev libgtest-dev libgflags-dev python3-dev libsqlite3-dev
cd /usr/src/googletest && sudo cmake . && sudo cmake --build . --target install
cd nvblox && cd nvblox && mkdir build && cd build
cmake .. -DPRE_CXX11_ABI_LINKABLE=ON
make -j32
sudo make install

  1. If you get a thrust error in esdf_integrator.cu during compilation, you need to recompile nvblox with ABI compatibility.

  2. If you get a C++ standard error, change CMAKE_CXX_STANDARD in line 5 of nvblox_torch/cpp/CMakeLists.txt to the specific standard from the error.

  3. Install nvblox_torch in your python environment:

git clone https://github.com/nvlabs/nvblox_torch.git
cd nvblox_torch
sh install.sh

Installing nvblox with Isaac Sim

Upgrade cmake with the below steps,

wget https://cmake.org/files/v3.19/cmake-3.19.5.tar.gz
tar -xvzf cmake-3.19.5.tar.gz
sudo apt install build-essential checkinstall zlib1g-dev libssl-dev
cd cmake-3.19.5 && ./bootstrap
make -j8
sudo make install

Install gflags and glog,

git clone https://github.com/gflags/gflags.git
cd gflags && mkdir build && cd build
ccmake .. && make & sudo make install

git clone https://github.com/google/glog.git
cd glog
cmake -S . -B build -G "Unix Makefiles"
cmake --build build --target install

sudo ln -s /usr/local/lib/libglog.so.0.7.0 /usr/local/lib/libglog.so.0

Install nvblox,

cmake .. -DPRE_CXX11_ABI_LINKABLE=ON -DCMAKE_CUDA_FLAGS="-gencode=arch=compute_70,code=sm_70" -DCMAKE_CUDA_ARCHITECTURES="70" -DSTDGPU_CUDA_ARCHITECTURE_FLAGS_USER="70" -DBUILD_FOR_ALL_ARCHS=ON

Install nvblox_torch,

export TORCH_CUDA_ARCH_LIST=7.0+PTX
sh install_isaac_sim.sh $(omni_python -c 'import torch.utils; print(torch.utils.cmake_prefix_path)')
omni_python -m pip install -e .

If you get a library not found runtime error, run export LD_LIBRARY_PATH=/usr/local/lib

nvblox troubleshooting

If you get a GLU not found runtime error with nvblox, install it with sudo apt install freeglut3-dev.

Generate Documentation

To generate documentation, pip install -e .[doc] --no-build-isolation and then run sphinx-build -b html docs public. The documentation can be viewed by opening public/index.html.
